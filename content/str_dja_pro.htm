Structure of a Djnago Project

1. manage.py
A command-line utility that lets you interact with this Django project. You use it to run the server, migrations, create apps, etc.
2. Project Folder (e.g., myproject/)
Contains the main project settings and configuration files:
  • __init__.py - Marks this directory as a Python package.
  • settings.py - Configuration for the entire project (database, installed apps, middleware, etc.).
  • urls.py - URL declarations for this project; routes URLs to views.
  • wsgi.py - Entry point for WSGI-compatible web servers to serve your project.
  • asgi.py - Entry point for ASGI-compatible servers (for async support).
3. App Folder(s) (e.g., myapp/)
Each app contains specific functionality. A project can have multiple apps.
Typical files inside an app:
  • models.py - Defines data models.
  • views.py - Contains the views (business logic).
  • urls.py (optional) - App-specific URL routing.
  • admin.py - Registers models with the Django admin.
  • apps.py - App configuration.
  • tests.py - Test cases for the app.
  • migrations/ - Database migration files.
  • templates/ (optional) - HTML templates related to the app.
  • static/ (optional) - Static files like CSS, JS, images.
4. Other Important Files
  • db.sqlite3 - Default SQLite database file (if used).
  • requirements.txt - Lists Python dependencies.
  • README.md - Project documentation.
  • .gitignore - Git ignore rules.
